## 这不是又一个 PPT 模板

当你对一个 AI 说「帮我做个PPT」时，99% 的情况它会从零写一段 python-pptx 代码，生成一个毫无设计感的白底黑字文档——然后你花 4 轮反馈把它改到勉强能用。

**PPT 大师** 是一个自带决策能力的 AI 工作台。它不生成 PPT，它**选择生成路径**：从 22 个已经过真实用户验证的开源技能中，匹配你当前场景的最优方案。

> 核心理念：做 PPT 不是写代码，是选方案。同一份大纲，选项选错输出差 10 倍。

---

## 它解决的问题

AI 做 PPT 有几个根深蒂固的问题，不是靠多写几行代码能解决的：

| 🎨 AI 没有视觉反馈回路 | 🧩 审美是统计出来的 |
|---|---|
| AI 看不到自己生成的页面长什么样，全靠 CSS 在脑子里「想象」。设计师看一眼就知道不对的东西，AI 要跑 3 轮反馈才能发现。 | AI 知道「黑底金字好看」是因为训练数据里大量高级海报这么配。但具体到留白多少、字距多大——没有身体感知。 |

| 🔁 每次从零开始 | 📊 工具多但散乱 |
|---|---|
| 前一轮的教训不会自动带入下一轮。这次「字太小」的修正，下次做新 PPT 还会再犯。 | GitHub 上有几十个 PPT 相关的 AI 技能（16k⭐的 guizang、27k⭐的 ppt-master、12.6k⭐的 MiniMax…），但没人告诉你什么场景用哪个。 |

## 核心架构 — 三层决策引擎

PPT 大师不是「又一个工具」——它是一个**元工具**（meta-tool），专门用来选择工具的。架构如下：

```
┌─────────────────────────────────────────────────┐
│  用户输入："帮我做个介绍短剧大师的 PPT"           │
│                                                  │
│  ▼ Layer 1 · 方案选择树                          │
│  ┌──────────────────────────────────────┐        │
│  │ 判断 9 个维度：格式·设计感·时效·修改量· │        │
│  │ 内容来源·协作需求·预算·输出平台·设备     │        │
│  └──────────────────────────────────────┘        │
│       │                                           │
│  ▼ Layer 2 · 路径执行                            │
│  路径A: 风格勘探（ppt-skill）                     │
│  路径B: 竖屏影视风（定制HTML→PNG）               │
│  路径C: SVG串行→PPTX（hugohe3）                  │
│  路径D: 企业级PPTX（MiniMax PptxGenJS）           │
│  路径E: 命令行操作（OfficeCLI）                   │
│  路径F~I: 极速/交互/演讲/协作                    │
│       │                                           │
│  ▼ Layer 3 · 设计系统 + QA                       │
│  ┌──────────────────────────────────────┐        │
│  │ 竖屏规范(9级字号体系+5层布局)           │        │
│  │ 横屏规范(16:9尺寸+主题契约+配色方案)    │        │
│  │ QA清单(通用+HTML+竖屏+PPTX 4类22项)     │        │
│  │ 错误模式记录(6轮反馈提炼15条避坑指南)   │        │
│  └──────────────────────────────────────┘        │
│       │                                           │
│  ▼ 输出：HTML / PPTX / PNG / PDF                 │
└─────────────────────────────────────────────────┘
       三层决策引擎：选择 → 执行 → 品控
```

## 为什么设计系统是「长出来」的

大多数 PPT 设计系统是自上而下定义的——设计师拍脑袋决定配色、字号、间距。PPT 大师的设计系统是**自下而上长出来的**。

短剧大师朋友圈海报在 **6 轮用户反馈**中不断收敛参数：

### 第1轮 · 初版
**字号用的 vw 单位，手机上只有 6px**
用户说「字太小了」。→ 固定 px 单位，正文必须≥22px。

### 第2轮 · 改字号
**用 absolute 定位，元素重叠**
用户说「有重叠」。→ 改用 flex flow 自然排列。

### 第3轮 · 改布局
**四周留白 60px，内容不够饱满**
用户说「留白太多」。→ padding 从 60px 降到 40px。

### 第4轮 · 缩留白
**品牌名 96px，不够震撼**
用户说「字体还不够大」。→ 品牌名升到 110px，案例数字升到 90px。

### 第5轮 · 扩字号
**逻辑层次不清晰**
用户说「逻辑性更强一些」。→ 5 层结构（品牌→案例→数据→优势→更多能力），每层用 border 分隔。

### 第6轮 · 定稿
**验收通过**
用户说「效果比之前好很多」。9 级字号体系、5 层布局模板、flex flow 排列、固定 px 字号、40px 内边距——全部固化到规则中。

> 每次否定不是失败，是规则收敛的一步。从用户的 6 轮反馈中，我们提炼出了 15 条硬约束——它们不是「设计建议」，而是被验证过的避坑指南。

## 方法论溯源暗线 — 每条规则都有来源

PPT 大师的每个决策点都标注了来源——不是为了显得严谨，而是为了防止未来使用时丢失上下文：

| 规则 | 来源 | 形成原因 |
|------|------|---------|
| 不要从零发明 | ← 短剧大师海报4轮失败 | python-pptx→pptxgenjs→Keynote→百度AI，最终定制HTML才通过 |
| Show Don't Tell | ← xiongy26/ppt-skill | 「先出3预览让用户看图选」比描述靠谱 |
| HTML > PPTX（设计感） | ← guizang + html-ppt 实测 | 浏览器渲染的设计感远超 PptxGenJS |
| 串行不跳步 | ← hugohe3/ppt-master 工作流 | 需求→方案→生成→自检→交付，跳过就是返工 |
| 竖屏规范 5 层结构 | ← 6 轮用户反馈 | 品牌→案例→数据→优势→更多能力，逻辑递进 |
| 字号固定 px | ← 第 1 轮反馈 | vw/clamp 手机上太小，固定 22-24px 才看清 |
| padding ≤ 40px | ← 第 3 轮反馈 | 60px 留白太多，内容不饱满 |
| flex flow 不用 absolute | ← 第 2 轮反馈 | absolute 定位在不同分辨率下重叠 |
